# 칸반 보드 구현 계획

## Context

반응형 칸반 보드를 순수 HTML/CSS/JS로 구현한다. 파일은 `index.html`, `style.css`, `app.js`로 분리하며 현재 디렉토리에 저장한다.

---

## 사용자 선택 사항

| 항목 | 선택 |
|------|------|
| 디자인 테마 | 클린 라이트 테마 (배경 #f0f2f5, 카드 #fff, 강조 #0052cc / #00875a / #de350b) |
| 카드 구성 | 풀 피처 (제목 + 설명 + 마감일 + 우선순위) |
| 애니메이션 | 다이나믹 풀 이펙트 (물리 기반 회전, 컨페티, 팝인) |
| 추가 기능 | 로컬 스토리지 저장, 카드 검색/필터, 컬럼별 카드 수 뱃지, 다크/라이트 모드 토글 |

---

## 파일 구조

```
kanban_panning/
├── PLAN.md           # 구현 계획 (현재 파일)
├── PRD.md            # 제품 요구사항 문서
├── TRD.md            # 기술 요구사항 문서
├── UserFlow.md       # 사용자 플로우
├── DatabaseDesign.md # 데이터 구조 설계
├── DesignSystem.md   # 디자인 시스템
├── TASKS.md          # 구현 태스크 목록
├── index.html        # 마크업 + 컴포넌트 구조
├── style.css         # 레이아웃, 테마, 애니메이션
└── app.js            # 드래그앤드롭, 카드 CRUD, 로컬스토리지, 이벤트
```

---

## 구현 순서

1. **index.html** — 마크업, 모달, 캔버스 포함 기본 골격
2. **style.css** — CSS 변수, 레이아웃, 카드 스타일, 애니메이션 정의
3. **app.js** — 데이터 로직 (CRUD, 스토리지) → 렌더링 → 드래그앤드롭 → 검색 → 컨페티 → 테마

---

## 검증 방법

- 브라우저에서 `index.html` 직접 열기 (서버 불필요, 순수 정적)
- 체크리스트:
  - [ ] 카드 추가 → 저장 → 로컬스토리지에 반영
  - [ ] 카드 드래그 → 컬럼 간 이동
  - [ ] Done 드롭 시 컨페티 이펙트 실행
  - [ ] 검색어 입력 시 카드 필터링
  - [ ] 다크모드 토글 후 새로고침해도 유지
  - [ ] 모바일 너비(375px)에서 레이아웃 정상
