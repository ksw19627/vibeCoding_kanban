# PRD — Product Requirements Document
## 칸반 보드 (Kanban Board)

---

## 1. 개요

### 1.1 제품 목적
개인 또는 소규모 팀이 작업을 시각적으로 관리할 수 있는 웹 기반 칸반 보드. 별도 서버나 설치 없이 브라우저만으로 즉시 사용 가능하다.

### 1.2 대상 사용자
- 개인 작업 관리가 필요한 사람
- 간단한 팀 프로젝트 진행을 시각화하려는 사람
- 복잡한 도구 없이 가볍게 칸반을 사용하고 싶은 개발자/학습자

### 1.3 기술 스택
- 순수 HTML5 / CSS3 / Vanilla JS
- 외부 라이브러리 없음
- 데이터 저장: `localStorage` (서버 불필요)

---

## 2. 기능 요구사항

### 2.1 핵심 기능 (Must Have)

#### F-01. 칸반 컬럼 3개 고정
| 컬럼 ID | 컬럼 이름 | 의미 |
|---------|-----------|------|
| `todo` | TO-DO | 예정된 작업 |
| `inprogress` | In Progress | 진행 중인 작업 |
| `done` | Done ✓ | 완료된 작업 |

#### F-02. 카드 생성
- 필수: 제목 (최대 60자)
- 선택: 설명, 마감일, 우선순위
- 모달 폼을 통해 입력
- 제목 미입력 시 저장 불가 (유효성 검사)

#### F-03. 카드 편집
- 기존 카드 클릭 → 편집 모달 열기
- 모든 필드 수정 가능

#### F-04. 카드 삭제
- 카드 내 삭제 버튼 → 확인 다이얼로그 → 삭제

#### F-05. 드래그 앤 드롭 이동
- HTML5 Drag and Drop API 사용
- 카드를 다른 컬럼으로 드래그하여 이동
- 컬럼 내 순서도 드래그로 변경 가능

#### F-06. 로컬 스토리지 영속성
- 카드 추가/수정/삭제/이동 시 즉시 `localStorage` 저장
- 페이지 새로고침 후에도 데이터 유지

---

### 2.2 확장 기능 (Should Have)

#### F-07. 카드 수 뱃지
- 각 컬럼 헤더에 현재 카드 수 표시
- 카드 변경 시 실시간 업데이트

#### F-08. 카드 검색 / 필터
- 상단 검색창에 텍스트 입력
- 제목 또는 설명에 검색어가 포함된 카드만 강조
- 미매칭 카드는 투명도를 낮춰 시각적으로 구분

#### F-09. 다크 / 라이트 모드 토글
- 우측 상단 버튼으로 테마 전환
- 선택된 테마를 `localStorage`에 저장하여 다음 방문 시 유지

---

### 2.3 UX 요구사항

#### F-10. 애니메이션 이펙트
| 이벤트 | 이펙트 |
|--------|--------|
| 카드 추가 | 팝인 애니메이션 (scale 0.6 → 1) |
| 드래그 시작 | 카드 3° 회전 + 그림자 강화 + 반투명 |
| 드래그 오버 | 대상 컬럼 점선 테두리 + 펄스 배경 |
| Done 컬럼 드롭 | 캔버스 컨페티 이펙트 (2초) |
| 모달 열기 | 페이드인 + 슬라이드업 |

#### F-11. 반응형 레이아웃
- 데스크탑(≥769px): 3열 가로 배치
- 태블릿/모바일(≤768px): 세로 스택 배치
- 모바일 최소 지원 너비: 375px

## 3. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 성능 | 카드 50개 기준 렌더링 100ms 이내 |
| 접근성 | 키보드 모달 닫기 (Esc), 폼 레이블 연결 |
| 브라우저 지원 | Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ |
| 의존성 | 외부 라이브러리 0개 |
| 설치 | 불필요 — 브라우저에서 index.html 직접 열기 |

---

## 4. 제외 범위 (Out of Scope)

- 서버 연동 / 클라우드 동기화
- 멀티 사용자 협업
- 첨부파일 업로드
- 컬럼 추가/삭제 커스터마이징
- 무한 스크롤 / 페이지네이션
